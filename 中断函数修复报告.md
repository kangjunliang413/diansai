# 中断函数修复报告 - Default_Handler 死循环问题

## 问题描述

程序在开启 `MOTOR_PID_INST_INT_IRQN` 中断后，单步调试直接掉进了 `Default_Handler` 的死循环。

## 根本原因

**中断服务函数名与启动文件向量表不匹配**

### 硬件映射关系

根据 SysConfig 配置文件 `Debug/ti_msp_dl_config.h:131-133`：

```c
#define PID_INST                    (TIMA0)
#define PID_INST_IRQHandler         TIMA0_IRQHandler
#define PID_INST_INT_IRQN           (TIMA0_INT_IRQn)
```

**映射链：**
```
MOTOR_PID_INST → PID_INST → TIMA0 (Timer A0)
```

### 问题代码

**错误的中断函数名（motor.c:282 原版本）：**
```c
void MOTOR_PID_INST_IRQHandler(void)  // ❌ 错误！这个名称不在向量表中
{
    switch (DL_Timer_getPendingInterrupt(MOTOR_PID_INST))
    {
    case DL_TIMER_IIDX_LOAD:
        // ...
        break;
    }
}
```

### 为什么会跳转到 Default_Handler？

1. **启动文件向量表**中定义的中断向量名称是 `TIMA0_IRQHandler`
2. 当 TIMA0 中断触发时，CPU 从向量表查找 `TIMA0_IRQHandler` 函数地址
3. 代码中使用的是 `MOTOR_PID_INST_IRQHandler`，向量表找不到此函数
4. 向量表中未定义的中断向量会指向 `Default_Handler`（默认弱链接）
5. 程序跳转到 `Default_Handler` 的死循环：
   ```c
   void Default_Handler(void)
   {
       while(1);  // 死循环
   }
   ```

## 修复方案

### 修改内容

**motor.c:289 - 正确的中断函数名：**
```c
void TIMA0_IRQHandler(void)  // ✅ 正确！与向量表匹配
{
    switch (DL_Timer_getPendingInterrupt(MOTOR_PID_INST))
    {
    case DL_TIMER_IIDX_LOAD:
        // 左轮闭环控制
        calculate_speed(1);      // 计算左轮速度
        motor_pi_loop(1);        // 执行左轮PI控制

        // 右轮闭环控制
        calculate_speed(2);      // 计算右轮速度
        motor_pi_loop(2);        // 执行右轮PI控制

        break;

    default:
        break;
    }
}
```

### 中断标志位清除验证

**已确认正确：**
- `DL_Timer_getPendingInterrupt(MOTOR_PID_INST)` 函数会**自动读取并清除** TIMA0 的中断标志位（IIDX 寄存器）
- 使用 `switch-case` 结构处理不同的中断源（`DL_TIMER_IIDX_LOAD`）
- **不需要**额外调用 `DL_Timer_clearInterruptStatus()` 等函数

### TI MSPM0 中断标志位清除机制

根据 TI DriverLib 文档：
```
DL_Timer_getPendingInterrupt() 
- 读取中断索引寄存器 (IIDX)
- 读取操作会自动清除对应的中断标志位
- 返回值指示触发的中断源
```

## 修复后的完整双轮 PI 控制代码

### 文件：user_driver/motor.c

**关键中断服务函数（289-307行）：**

```c
/**
 * @brief PID定时器中断服务函数（TIMA0）
 *
 * 硬件映射：
 * - MOTOR_PID_INST → PID_INST → TIMA0
 * - 中断函数名必须为：TIMA0_IRQHandler（与启动文件向量表匹配）
 *
 * 触发周期：50ms（20Hz）
 *
 * 完整闭环流程：
 * 1. 读取编码器脉冲增量
 * 2. 计算实际物理速度
 * 3. 执行PI算法计算PWM
 * 4. 更新电机PWM输出
 *
 * 中断标志位清除：
 * DL_Timer_getPendingInterrupt() 会自动读取并清除中断标志位（IIDX寄存器）
 */
void TIMA0_IRQHandler(void)
{
    switch (DL_Timer_getPendingInterrupt(MOTOR_PID_INST))
    {
    case DL_TIMER_IIDX_LOAD:
        // 左轮闭环控制
        calculate_speed(1);      // 计算左轮速度
        motor_pi_loop(1);        // 执行左轮PI控制

        // 右轮闭环控制
        calculate_speed(2);      // 计算右轮速度
        motor_pi_loop(2);        // 执行右轮PI控制

        break;

    default:
        break;
    }
}
```

### 关键经验教训

1. **中断函数名必须严格匹配启动文件向量表**
   - 查看 SysConfig 生成的 `ti_msp_dl_config.h` 中的 `XXX_IRQHandler` 定义
   - 不要使用自定义宏名称作为中断函数名

2. **TI MSPM0 中断标志位清除机制**
   - `DL_Timer_getPendingInterrupt()` 和 `DL_GPIO_getPendingInterrupt()` 会自动清除标志位
   - 不需要手动调用 clear 函数

3. **调试技巧**
   - 如果程序跳转到 `Default_Handler`，首先检查中断函数名是否正确
   - 在启动文件中搜索向量表，确认标准中断函数名
